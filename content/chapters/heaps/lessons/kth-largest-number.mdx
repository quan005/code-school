<ProblemStatement title="Find the kth largest, not all of them">
  Given an array of numbers <code>nums</code> and a number <code>k</code>,
  return the <strong>kth</strong> largest number.
</ProblemStatement>

## Example

```txt
nums = [3, 2, 1, 5, 6, 4], k = 2
answer = 5
```

<Intuition title="The smallest of the top k is the kth largest">
  If a min-heap stores exactly the <code>k</code> largest values, then the top
  of that heap is the smallest of those survivors. That makes it the kth largest
  overall.
</Intuition>

<InterviewTip>
  Top-k and kth are close cousins. Many kth problems reuse the same heap idea as
  top-k problems.
</InterviewTip>

## TypeScript Solution

```ts
function kthLargest(nums: number[], k: number): number {
  const minHeap = new Heap<number>((a, b) => a < b);

  for (const num of nums) {
    minHeap.push(num);

    if (minHeap.size() > k) {
      minHeap.pop();
    }
  }

  return minHeap.peek()!;
}
```
