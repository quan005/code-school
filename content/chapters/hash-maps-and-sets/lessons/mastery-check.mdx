<ProblemStatement title="Final chapter check">
  This mastery check asks you to choose the right tool, explain your reasoning,
  and finish one last implementation task from memory.
</ProblemStatement>

<Intuition title="What strong answers sound like">
  Strong answers do more than say “map” or “set.” They explain what the problem
  needs to remember and why that makes one tool a better fit.
</Intuition>

<HintDrawer title="Still unsure?">
  Use a set for presence. Use a map for counts, indexes, and matches.
</HintDrawer>

## Part A: Multiple Choice

- Which lesson is a set problem because it only cares about seen-before
  checking?
- Which lesson is a map problem because it needs counts?
- Which lesson uses a set to compare two collections?

## Part B: Short Answer

- What is the difference between a set and a map?
- Why is Two Sum a map problem?
- Why does First Unique Character count first and search second?
