<ProblemStatement title="Maps remember, sets notice">
  A <strong>hash map</strong> remembers extra information about something. A{" "}
  <strong>set</strong> only remembers whether that thing has been seen.
</ProblemStatement>

<Intuition title="Picture cubbies and a sticker board">
  A map is like classroom cubbies with labels. The label is the key, and the
  stuff inside is the value. A set is like a sticker board that only keeps one
  copy of each sticker no matter how many times you add it.
</Intuition>

<InterviewTip>
  Ask one question first: do I only care whether the thing exists, or do I need
  extra information like a count, an index, or a match?
</InterviewTip>

## Four Main Strategies

- <strong>Seen-before checking</strong> uses a set.
- <strong>Counting</strong> uses a map.
- <strong>Lookup by need</strong> uses a map.
- <strong>Matching between collections</strong> uses a set or a map.

## Real-world picture

A teacher can use:

- a <strong>set</strong> to track who already checked in
- a <strong>map</strong> to track how many times each student visited the
  reading station

That is exactly how these tools work in code.

## Tiny Practice

- If the problem asks, “How many times did each letter appear?” use a{" "}
  <strong>map</strong>.
- If the problem asks, “Have I seen this before?” use a <strong>set</strong>.

## Lesson Summary

- Use a set when you only care if something exists.
- Use a map when you need counts, indexes, or matches.
- These tools often replace slow nested loops with faster lookups.
