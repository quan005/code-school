<ProblemStatement title="One pattern, many clues">
  Fast and slow pointers use the same speed difference idea in several kinds of
  problems: middles, cycles, repeated loops, and half comparisons.
</ProblemStatement>

<Intuition title="What the pattern tells us">
  If fast reaches null, the path ends. If fast meets slow, there is a loop. If
  fast finishes while slow is still moving, slow is around the middle.
</Intuition>
