<ProblemStatement title="Find where the number belongs">
  Given a sorted array of distinct integers and a target, return the index if
  the target is found. If not, return the index where it should be inserted so
  the array stays sorted.
</ProblemStatement>

## Example

```txt
nums = [1, 3, 5, 7], target = 4
answer = 2
```

<Intuition title="If it is not there, return the landing spot">
  This feels like normal binary search, but we do not return <code>-1</code> at
  the end. When the loop stops, <code>left</code> lands exactly where the target
  should go.
</Intuition>

<InterviewTip>
  Insert-position problems are a clue that binary search is really searching for
  a place, not only for an exact match.
</InterviewTip>

## TypeScript Solution

```ts
function searchInsert(nums: number[], target: number): number {
  let left = 0;
  let right = nums.length - 1;

  while (left <= right) {
    const mid = Math.floor((left + right) / 2);

    if (nums[mid] === target) {
      return mid;
    } else if (nums[mid] < target) {
      left = mid + 1;
    } else {
      right = mid - 1;
    }
  }

  return left;
}
```
