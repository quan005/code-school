<ProblemStatement title="Find the first copy">
  Given a sorted array of integers <code>nums</code> that may have duplicates
  and a target, return the index of the <strong>first</strong> occurrence of the
  target. If the target does not exist, return <code>-1</code>.
</ProblemStatement>

## Example

```txt
nums = [1, 2, 2, 2, 4, 5], target = 2
answer = 1
```

<Intuition title="Finding one copy is not enough">
  Normal binary search can stop when it finds the target. This problem cannot.
  After finding a copy, we save that index and keep searching left in case there
  is an earlier copy.
</Intuition>

<InterviewTip>
  A common binary-search twist is: save the answer, then keep shrinking toward
  the side that might still hold a better answer.
</InterviewTip>

## TypeScript Solution

```ts
function firstOccurrence(nums: number[], target: number): number {
  let left = 0;
  let right = nums.length - 1;
  let answer = -1;

  while (left <= right) {
    const mid = Math.floor((left + right) / 2);

    if (nums[mid] === target) {
      answer = mid;
      right = mid - 1;
    } else if (nums[mid] < target) {
      left = mid + 1;
    } else {
      right = mid - 1;
    }
  }

  return answer;
}
```
