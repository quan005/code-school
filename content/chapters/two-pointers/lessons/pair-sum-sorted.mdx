<ProblemStatement title="Find the target pair">
  Given a sorted array of numbers and a target sum, return the indexes of any
  pair that adds up to the target. If no pair exists, return an empty array.
</ProblemStatement>

## Examples

```txt
nums = [-5, -2, 3, 4, 6], target = 7 -> [2, 3]
nums = [1, 1, 1], target = 2 -> [0, 1]
```

## Constraints

- The array is already sorted in ascending order.
- You need two different indexes.
- Any correct pair is okay.

<Intuition title="Use the sorted order as a clue">
  When the list is sorted, the left pointer points to a smaller number and the
  right pointer points to a bigger number. If the total is too small, only the
  left side can help make it bigger. If the total is too big, only the right
  side can help make it smaller.
</Intuition>

<InterviewTip>
  This is a good two-pointer problem because every pointer move removes many
  impossible pairs at once.
</InterviewTip>

## Brute Force vs. Better Movement

The slow idea checks every pair with nested loops. That works, but it can take
about `O(n²)` time.

The smarter idea starts with `left` at the beginning and `right` at the end:

- if the sum is too small, move `left`
- if the sum is too big, move `right`
- if the sum matches, return the answer

## Walkthrough

For `nums = [-5, -2, 3, 4, 6]` and `target = 7`:

1. Start with `left = 0` and `right = 4`. The sum is `1`, so move `left`.
2. Now the sum is `4`, so move `left` again.
3. Now the sum is `9`, so move `right`.
4. The sum becomes `7`, so the answer is `[2, 3]`.

<HintDrawer title="Need a move rule?">
  If `nums[left] + nums[right]` is smaller than the target, move `left`. If it
  is bigger than the target, move `right`.
</HintDrawer>
