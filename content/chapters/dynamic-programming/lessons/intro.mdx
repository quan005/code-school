<ProblemStatement title="Dynamic programming saves smaller answers so bigger answers get easier">
  Dynamic programming, often called <code>DP</code>, solves a big problem by
  using answers to smaller repeated problems and saving those smaller answers.
</ProblemStatement>

<Intuition title="Think about climbing stairs one step at a time">
  If you already know the answer for step 4 and step 5, then step 6 can become
  easy. DP is the idea of saving those smaller answers instead of recomputing
  them again and again.
</Intuition>

<InterviewTip>
  A <code>base case</code> is one of the first tiny answers we already know for
  sure. <code>Overlapping subproblems</code> means the same smaller question
  shows up more than once.
</InterviewTip>

## Helpful DP words

- <strong>base case</strong>: the first small answer we already know
- <strong>overlapping subproblems</strong>: the same smaller question repeats
- <strong>bottom-up</strong>: start small and build larger answers
- <strong>top-down</strong>: start big, recurse smaller, and save answers

## Strong DP clues

- number of ways
- minimum cost
- maximum score
- answer depends on earlier positions
- the same smaller question repeats

## Lesson Summary

- DP saves smaller answers and reuses them later.
- A base case is a tiny known answer.
- Overlapping subproblems mean the same small work repeats.
- Bottom-up DP builds from the smallest answers upward.
- DP clues often involve minimum, maximum, or number of ways.
