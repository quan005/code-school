<ProblemStatement title="Make as many children happy as possible">
  Each child has a greed factor and each cookie has a size. A child is happy if
  the cookie size is at least the greed factor. Return the maximum number of
  happy children.
</ProblemStatement>

## Example

```txt
g = [1, 2, 3]
s = [1, 1]
answer = 1
```

<Intuition title="Do not waste a big cookie on a small need">
  Sort the children from least greedy to most greedy. Sort the cookies from
  smallest to biggest. Then give the smallest cookie that can still satisfy the
  current child.
</Intuition>

<InterviewTip>
  This is a classic greedy matching idea: use the smallest helpful resource so
  the bigger resources stay available for bigger needs later.
</InterviewTip>

## TypeScript Solution

```ts
function findContentChildren(g: number[], s: number[]): number {
  g.sort((a, b) => a - b);
  s.sort((a, b) => a - b);

  let child = 0;
  let cookie = 0;
  let happy = 0;

  while (child < g.length && cookie < s.length) {
    if (s[cookie] >= g[child]) {
      happy += 1;
      child += 1;
      cookie += 1;
    } else {
      cookie += 1;
    }
  }

  return happy;
}
```
