<ProblemStatement title="A prefix sum stores the total so far">
  A prefix sum stores the running total from the beginning up to each position.
</ProblemStatement>

<Intuition title="Think about adding as you walk">
  If the numbers are <code>[2, 4, 1, 3]</code>, the running totals become
  <code>[2, 6, 7, 10]</code>. Each new total remembers everything from the start
  up to that spot.
</Intuition>

<InterviewTip>
  Prefix sums are a clue when problems ask many range sums, cumulative totals,
  left sum versus right sum, or repeated “sum from here to there” questions.
</InterviewTip>

<ComplexityCard
  time="Build once in O(n), then many questions may become O(1)"
  space="Often O(n) for the helper array"
  notes="The big idea is precomputing: do a little setup once so later work becomes faster."
/>

## Helpful formula

To get the sum from <code>left</code> to <code>right</code>:

- use <code>prefix[right]</code>
- subtract the total before <code>left</code>

## Lesson Summary

- A prefix sum stores the running total from the beginning.
- It helps answer later range questions quickly.
- Precomputing means doing setup once to save time later.
- Running totals are one of the main clues for this pattern.
