<ProblemStatement title="Find the target in a rotated sorted array">
  Given a sorted array that was rotated at some unknown point, return the index
  of the target if it exists. Otherwise, return <code>-1</code>.
</ProblemStatement>

## Example

```txt
nums = [4, 5, 6, 7, 0, 1, 2]
target = 0
answer = 4
```

<Intuition title="Even after rotation, one side is still ordered">
  At any middle point, at least one half is still sorted. That sorted half gives
  enough information to decide where the target can or cannot be.
</Intuition>

<InterviewTip>
  This is still a search-with-order problem. The order looks partly broken, but
  one side stays trustworthy at each step.
</InterviewTip>

## TypeScript Solution

```ts
function searchRotated(nums: number[], target: number): number {
  let left = 0;
  let right = nums.length - 1;

  while (left <= right) {
    const mid = Math.floor((left + right) / 2);

    if (nums[mid] === target) {
      return mid;
    }

    if (nums[left] <= nums[mid]) {
      if (nums[left] <= target && target < nums[mid]) {
        right = mid - 1;
      } else {
        left = mid + 1;
      }
    } else {
      if (nums[mid] < target && target <= nums[right]) {
        left = mid + 1;
      } else {
        right = mid - 1;
      }
    }
  }

  return -1;
}
```
