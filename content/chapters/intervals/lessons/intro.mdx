<ProblemStatement title="An interval is a range">
  An interval is a pair like <code>[start, end]</code>. It shows where something
  begins and where it stops.
</ProblemStatement>

<Intuition title="Think about a school schedule">
  A class from 9:00 to 10:00 is one interval. A class from 9:30 to 11:00 is
  another. If the time blocks share space, they overlap.
</Intuition>

<InterviewTip>
  Interval problems often use pairs like <code>[start, end]</code> and ask about
  overlap, merge, insert, conflicts, rooms, or schedules.
</InterviewTip>

  time="Many interval problems start with O(n log n) sorting"

## Two big questions

- Do these intervals overlap?
- If they overlap, what should we do next?

## Helpful overlap rule

Two intervals overlap when the next one starts before or at the current end.

## Lesson Summary

- An interval has a start and an end.
- Many interval problems ask about overlap or gaps.
- Sorting by start time often helps a lot.
- Schedules and ranges are common interval clues.
