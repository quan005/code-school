<ProblemStatement title="Build the word tree one letter at a time">
  Design a trie that can insert words. Each inserted word should follow existing
  shared letters when possible and create new nodes only when needed.
</ProblemStatement>

## Example

```txt
insert("cat")
insert("car")
```

The path for <code>c</code> and <code>a</code> is shared. The last letter
splits into <code>t</code> and <code>r</code>.

<Intuition title="Reuse what is already there">
  Start at the root. For each letter, check whether that next letter node
  already exists. If it does, reuse it. If it does not, create it. After the
  last letter, mark <code>isWord = true</code>.
</Intuition>

<InterviewTip>
  In trie problems, the end marker matters. Just reaching the letter path is not
  enough to prove a full word exists.
</InterviewTip>

## TypeScript Solution

```ts
class TrieNode {
  children: Map<string, TrieNode>;
  isWord: boolean;

  constructor() {
    this.children = new Map();
    this.isWord = false;
  }
}

class Trie {
  root: TrieNode;

  constructor() {
    this.root = new TrieNode();
  }

  insert(word: string): void {
    let node = this.root;

    for (const letter of word) {
      if (!node.children.has(letter)) {
        node.children.set(letter, new TrieNode());
      }

      node = node.children.get(letter)!;
    }

    node.isWord = true;
  }
}
```

  time="O(m)"
  space="O(m) new nodes in the worst case"
  notes="Here, m means the number of letters in the word. We take about one step per letter."

<HintDrawer title="Only build what is missing">
  Each letter asks one question: does this next child already exist? If yes, use
  it. If no, create it.
</HintDrawer>
