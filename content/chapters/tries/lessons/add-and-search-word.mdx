<ProblemStatement title="Search words even when one letter is unknown">
  Design a structure that supports adding words and searching them, where
  <code>.</code> can match any one letter.
</ProblemStatement>

## Example

```txt
stored words: bad, dad, mad
search(".ad") -> true
search("b..") -> true
search("..z") -> false
```

<Intuition title="A dot means the path can branch">
  A normal letter gives only one next step. A dot means any child could work, so
  we try each child and keep going from there. That is why recursion fits well
  here.
</Intuition>

<InterviewTip>
  Wildcards often turn one straight search into a branching search. That is a
  recursion clue.
</InterviewTip>

## TypeScript Solution

```ts
search(word: string): boolean {
  const dfs = (index: number, node: TrieNode): boolean => {
    if (index === word.length) {
      return node.isWord;
    }

    const letter = word[index];

    if (letter === ".") {
      for (const child of node.children.values()) {
        if (dfs(index + 1, child)) {
          return true;
        }
      }

      return false;
    }

    const nextNode = node.children.get(letter);
    if (!nextNode) {
      return false;
    }

    return dfs(index + 1, nextNode);
  };

  return dfs(0, this.root);
}
```
