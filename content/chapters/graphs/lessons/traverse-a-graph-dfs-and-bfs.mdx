<ProblemStatement title="Visit the connected nodes without getting stuck">
  Given a graph and a starting node, return the nodes you can visit by using DFS
  or BFS.
</ProblemStatement>

## Example Graph

```txt
A -- B
|    |
C -- D
```

<Intuition title="Graphs can loop, so we need memory">
  In a graph, you can come back to a node you already visited. That is why we
  keep a visited set. DFS goes down one route first. BFS looks at the nearby
  nodes first.
</Intuition>

<InterviewTip>
  If the graph can loop back, visited tracking is not optional. It protects you
  from walking in circles forever.
</InterviewTip>

## TypeScript Solution

```ts
function dfs(graph: Map<string, string[]>, start: string): string[] {
  const result: string[] = [];
  const visited = new Set<string>();

  function walk(node: string): void {
    if (visited.has(node)) {
      return;
    }

    visited.add(node);
    result.push(node);

    for (const neighbor of graph.get(node) ?? []) {
      walk(neighbor);
    }
  }

  walk(start);
  return result;
}

function bfs(graph: Map<string, string[]>, start: string): string[] {
  const result: string[] = [];
  const visited = new Set<string>([start]);
  const queue: string[] = [start];

  while (queue.length > 0) {
    const node = queue.shift()!;
    result.push(node);

    for (const neighbor of graph.get(node) ?? []) {
      if (!visited.has(neighbor)) {
        visited.add(neighbor);
        queue.push(neighbor);
      }
    }
  }

  return result;
}
```
