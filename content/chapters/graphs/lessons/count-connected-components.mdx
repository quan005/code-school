<ProblemStatement title="How many separate groups are there?">
  Given a graph, count how many connected components it has. A connected
  component is one group of nodes that can all reach one another.
</ProblemStatement>

## Example

```txt
A -- B    C -- D    E

answer = 3
```

<Intuition title="Each new unexplored island of nodes starts a new group">
  Walk through all nodes. When you find one that has not been visited yet, that
  means you found a new component. Start a traversal there, mark that whole
  group, and increase the count by one.
</Intuition>

<InterviewTip>
  Component problems often sound like "How many separate groups are there?"
</InterviewTip>

## TypeScript Solution

```ts
function countComponents(graph: Map<string, string[]>): number {
  const visited = new Set<string>();
  let count = 0;

  function dfs(node: string): void {
    if (visited.has(node)) {
      return;
    }

    visited.add(node);

    for (const neighbor of graph.get(node) ?? []) {
      dfs(neighbor);
    }
  }

  for (const node of graph.keys()) {
    if (!visited.has(node)) {
      count += 1;
      dfs(node);
    }
  }

  return count;
}
```

<HintDrawer title="New unvisited start means new group">
  If a node is still unvisited when you scan to it, no earlier traversal reached
  it. That means it belongs to a new component.
</HintDrawer>
