# Lesson Design Refresh Plan

This plan covers the next content-design pass for the full `Code Patterns` curriculum. It translates the sample-lesson model in [design-system.md](./design-system.md) into one redesign epic per chapter and one implementation story per lesson.

## Scope
- Focus: lesson experience redesign, not new algorithm coverage
- Source of truth: [design-system.md](./design-system.md) and the chapter curriculum docs under `docs/curriculum/code-patterns`
- Target outcome: every lesson should read like a concrete screen design with a scene, support strip, interaction flow, and named component usage

## Shared Story Definition of Done
Every lesson story is complete only when:
- the lesson has a clear habitat, mascot, and screen composition
- the lesson defines scene, support strip, action zone, and navigation
- the lesson documents visual details and interaction flow
- the lesson identifies the exact reusable components used on the screen
- the final lesson surface stays child-readable, low-pressure, and focused on one primary action
